## Authorization

This endpoint requires authentication — send your API key as a Bearer token.

<ResponseField name="Authorization" type="string" required>
  Example:

  `Authorization | Bearer Paste-Your-API-Key-Here`
</ResponseField>

## Query Parameters

<ParamField query="productId" type="string" required>
  The unique identifier of the product whose members you want to list. **Required** — if omitted the
  endpoint responds `400 "Required: productId"`.

  Example:

  `f5103e7c-9e30-467b-bc32-a96fdcb65849`
</ParamField>

<ParamField query="limit" type="integer" default="10">
  The maximum number of members to return per page. Defaults to `10` and is capped at `50` (a larger value
  is clamped to `50`).
</ParamField>

<ParamField query="cursor" type="string">
  The pagination cursor: a **unix epoch-ms string** (for example `"1718873457994"`) compared against each
  member's `createdAt`. Pass the `nextStartingAfter` / `prevStartingAfter` value returned in the previous
  response's `pagination` object to fetch the adjacent page. Note the request parameter is named `cursor`,
  while the response returns the boundary cursors under `nextStartingAfter` / `prevStartingAfter`.
</ParamField>

<ParamField query="direction" type="string" default="next">
  The pagination direction relative to `cursor`. One of `next` or `prev`. Defaults to `next`.
</ParamField>

<ParamField query="sortOrder" type="string" default="DESC">
  Sort order by `createdAt`. One of `ASC` or `DESC`. Defaults to `DESC` (newest first).
</ParamField>

<ParamField query="searchTerm" type="string">
  Free-text search filter. Case-insensitively matches against the member's `memberId`, `customer.name`, or
  `customer.email`.
</ParamField>

<ParamField query="startDate" type="string">
  Lower bound of the `createdAt` date range to filter members by (inclusive).
</ParamField>

<ParamField query="endDate" type="string">
  Upper bound of the `createdAt` date range to filter members by (inclusive).
</ParamField>

<ParamField query="isChurnedMember" type="boolean">
  When `true`, filters the result to churned members only — those whose `status` is `stopped` or `inactive` —
  and adds a top-level `totalMember` sibling key to the response. When omitted (or not truthy), no churn
  filter is applied and `totalMember` is **not** included.
</ParamField>

## Response

Successful Response

### Main Structure (Root Object)

<ResponseField name="statusCode" type="integer">
  Status code from the API. `200` on success.
</ResponseField>

<ResponseField name="messages" type="string">
  Status message that describes the status code. Note the **plural** `messages` key (read/list endpoints use
  `messages`; write endpoints use the singular `message`).
</ResponseField>

<ResponseField name="totalMember" type="integer">
  The total count of churned members. **Only present when `isChurnedMember=true`** — a top-level sibling of
  `data`; omitted entirely otherwise.
</ResponseField>

<ResponseField name="data" type="array of object">
  The list of membership members for the product, ordered by `createdAt` according to `sortOrder`. Each row
  uses **literal dotted-key** columns (flat string keys such as `membershipTier.name`), not nested objects.
</ResponseField>

<ResponseField name="data[].id" type="string">
  The unique identifier of the membership member record.
</ResponseField>

<ResponseField name="data[].createdAt" type="string">
  ISO 8601 timestamp (UTC) of when the member was created. This is the field the `cursor` paginates against.
</ResponseField>

<ResponseField name="data[].customerId" type="string">
  The unique identifier of the underlying customer.
</ResponseField>

<ResponseField name="data[].membershipTierId" type="string">
  The unique identifier of the member's membership tier.
</ResponseField>

<ResponseField name="data[].nextPayment" type="string">
  ISO 8601 timestamp (UTC) of the member's next scheduled payment.
</ResponseField>

<ResponseField name="data[].status" type="string">
  Current status of the member (for example `active`, `stopped`, `inactive`, `finished`).
</ResponseField>

<ResponseField name="data[].updatedAt" type="string">
  ISO 8601 timestamp (UTC) of when the member record was last updated.
</ResponseField>

<ResponseField name="data[].userId" type="string">
  The unique identifier of the owning user (merchant) the member belongs to.
</ResponseField>

<ResponseField name="data[].memberId" type="string">
  The public membership code for the member (for example `PUYSW40N`).
</ResponseField>

<ResponseField name="data[].membershipTier.name" type="string">
  The display name of the member's tier. This is a **literal dotted string key** in each row, not a nested
  `membershipTier` object.
</ResponseField>

<ResponseField name="data[].membershipTier.gracePeriodInDays" type="integer">
  The grace period (in days) for the member's tier. A **literal dotted string key**.
</ResponseField>

<ResponseField name="data[].customer.name" type="string">
  The customer's full name. A **literal dotted string key**.
</ResponseField>

<ResponseField name="data[].customer.mobile" type="string">
  The customer's mobile phone number. A **literal dotted string key**.
</ResponseField>

<ResponseField name="data[].customer.email" type="string">
  The customer's email address. A **literal dotted string key**.
</ResponseField>

### Pagination Object

<ResponseField name="pagination" type="object">
  Cursor-based pagination metadata.
</ResponseField>

<ResponseField name="pagination.limit" type="integer">
  The page size that was applied to this request (after defaulting to `10` / clamping to a maximum of `50`).
</ResponseField>

<ResponseField name="pagination.nextStartingAfter" type="string | null">
  The cursor (unix epoch-ms string) to pass back as `cursor` with `direction=next` to fetch the next page.
  `null` when there is no next page. Note this response key differs in name from the request param `cursor`.
</ResponseField>

<ResponseField name="pagination.prevStartingAfter" type="string | null">
  The cursor (unix epoch-ms string) to pass back as `cursor` with `direction=prev` to fetch the previous
  page. `null` when there is no previous page.
</ResponseField>

<ResponseField name="pagination.hasNextPage" type="boolean">
  Whether a next page of results exists.
</ResponseField>

<ResponseField name="pagination.hasPrevPage" type="boolean">
  Whether a previous page of results exists.
</ResponseField>
