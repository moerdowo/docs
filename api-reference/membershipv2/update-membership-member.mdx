## Authorization

This endpoint requires authentication — send your API key as a Bearer token.

<ResponseField name="Authorization" type="string" required>
  Example:

  `Authorization | Bearer Paste-Your-API-Key-Here`
</ResponseField>

## Path Parameters

<ParamField path="memberId" type="string" required>
  The public membership code of the member to update (for example `PUYSW40N`).

  Example:

  `PUYSW40N`
</ParamField>

## Request Body

`productId` is the only required field. The remaining fields are optional — send only the ones you want to
change.

<ParamField body="productId" type="string" required>
  The unique identifier of the product the member belongs to. The product must belong to the authenticated
  user. If missing the endpoint responds `400 "Required: productId"`.

  Example:

  `f5103e7c-9e30-467b-bc32-a96fdcb65849`
</ParamField>

<ParamField body="membershipTierId" type="string">
  The unique identifier of the membership tier to move the member to. Must reference an existing tier,
  otherwise the endpoint responds `400 "Membership tier tidak ditemukan."`.
</ParamField>

<ParamField body="membershipMonthlyPeriod" type="integer">
  The monthly billing period for the membership. Must reference an existing period, otherwise the endpoint
  responds `400 "periode membership tidak ditemukan"`.
</ParamField>

<ParamField body="status" type="string">
  The new status for the member (for example `active`, `stopped`, `inactive`, `finished`).
</ParamField>

<ParamField body="nextPayment" type="string">
  The member's next scheduled payment, as a **UTC ISO 8601 string** (for example
  `2026-08-20T09:10:57.994Z`) — not an epoch-millisecond value. An invalid value responds with a `400`.
</ParamField>

<ParamField body="expiredAt" type="string">
  The membership expiry time, as a **UTC ISO 8601 string** (for example `2026-09-20T09:10:57.994Z`). Must
  not be earlier than `nextPayment`, otherwise the endpoint responds
  `400 "expiredAt tidak boleh lebih awal dari nextPayment."`.
</ParamField>

## Response

Successful Response — HTTP `200`

<ResponseField name="message" type="string">

</ResponseField>

## Errors

All errors use the envelope `{ "statusCode": <code>, "message": "<reason>" }` (singular `message`).

```json Error Example 400
{
    "statusCode": 400,
    "message": "Required: productId"
}
```

<ResponseField name="statusCode" type="integer">
  The HTTP status code of the error.
</ResponseField>

<ResponseField name="message" type="string">
  A human-readable description of the error.
</ResponseField>

Common errors:

| Status | `message` | When |
| --- | --- | --- |
| `400` | `Required: productId` | `productId` is missing from the body (`memberId` comes from the path). |
| `400` | `You are not authorized to edit this product!` | The `productId` does not belong to the authenticated user. |
| `400` | `Membership tier tidak ditemukan.` | The supplied `membershipTierId` does not match an existing tier. |
| `400` | `periode membership tidak ditemukan` | The supplied `membershipMonthlyPeriod` does not match an existing period. |
| `400` | `expiredAt tidak boleh lebih awal dari nextPayment.` | `expiredAt` is earlier than `nextPayment`. |
| `400` | _\(invalid date\)_ | `nextPayment` or `expiredAt` is not a valid UTC ISO 8601 string. |
| `404` | `Member dengan ID <memberId> tidak ditemukan.` | No member with that `memberId` exists for the given `productId`. |
